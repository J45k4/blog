Value Shift in Software Development

Around 2025, when the first models that were truly good at agent work showed up, coding agents started to feel genuinely useful. Since then, the steady trend has been: better results every iteration. If that keeps going, then a GPT-5 level model will be almost free in one year, while the best models keep getting noticeably stronger.

A concrete example of the cost drop: about a year ago, o1 Standard API pricing was $15 / 1M input and $60 / 1M output. Today, GPT-5.2 Standard is listed at $1.75 / 1M input and $14 / 1M output. That is roughly 8x cheaper on input and 4x cheaper on output. Cached input went from $7.50 / 1M to $0.175 / 1M, which is about 40x cheaper.

If models can handle most routine coding tasks, the value in software development shifts. Writing code becomes a smaller part of the job. Defining the right problem and validating the solution become the center of gravity.

A Possible Workflow

Imagine a product-focused person who gathers customer needs and writes a clear ticket. Instead of handing it to an engineer first, they hand it to an AI system wired into the ticketing flow.

  1. AI makes changes, pushes a feature branch, and opens a merge request.
  2. If CI passes, the product person deploys to a dev environment.
  3. They test the feature and verify it matches the intent.
  4. If something is wrong, they report it to the AI and repeat.
  5. Once it works, a software engineer reviews for quality and consistency.
  6. The engineer merges it.

The Engineer Becomes a Taste Tester

The engineer role shifts toward taste, quality, and long-term maintainability. It is less about writing every line, and more about making sure the final result is solid.

A New Kind of Product Tester Role

The person driving the loop does not need deep programming knowledge. They need to know the product, test the behavior, and communicate with the agent effectively.

  • Comfortable using a computer and a terminal
  • Deep understanding of the product and its users
  • Good at describing issues and iterating with the AI

In that world, the value is less about typing code and more about defining the right thing, testing it, and knowing when it is good enough to ship.
